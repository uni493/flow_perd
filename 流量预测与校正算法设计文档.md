# 流量预测与校准算法设计文档

## 0. 指标结论

当前方案的目标是预测“从当前时间片到当天结束还会有多少流量”。离线实验说明：在基础预测上加入时间片校准后，误差明显下降。

| 方案 | 当前评估指标 |
| --- | ---: |
| 老方法 / 未校准基础预测 | `0.90` |
| 当天已有流量累计校准 | `0.75` |
| 最近 `{L}` 个时间片校准 | `0.70` |

这里的 `{L}` 是时间片窗口长度占位符，后续按实际实验窗口替换。

当前评估指标不是常见的“加权绝对百分比误差”（Weighted Absolute Percentage Error, WAPE），也不是“平均绝对百分比误差”（Mean Absolute Percentage Error, MAPE）。虽然 SQL 字段名里沿用了 `*_wape`，但实际口径是：

> 用当前时间片流量作为权重，计算剩余流量预测的加权相对误差。

公式为：

$$
\text{Score}
=
\frac{
\sum_s q_s \cdot \frac{|\hat{R}_s - R_s|}{R_s}
}{
\sum_s q_s
}
$$

其中 $R_s$ 是真实剩余流量，$\hat{R}_s$ 是预测剩余流量，$q_s$ 是当前时间片流量。这个指标越小越好。

因此，更准确的结论表述是：

> 在当前 SQL 评估口径下，最近 `{L}` 个时间片校准把指标从 `0.90` 降到 `0.70`，优于当天累计校准的 `0.75`。

---

## 1. 方案概览

一天被切成 `0-47` 共 48 个时间片。每到一个时间片开始前，系统需要估计某个任务从当前时间片到当天结束还会有多少流量。

方案分成三步：

1. 基础预测：先根据历史数据预测每个时间片应该有多少流量。
2. 时间片校准：看最近 `{L}` 个时间片里，真实流量比基础预测偏高还是偏低。
3. 剩余预测：用这个偏差去放大或缩小基础剩余流量预测。

核心公式是：

$$
\hat{R}
=
\hat{R}^{base}
\cdot
c
$$

其中：

| 名词 | 含义 |
| --- | --- |
| $\hat{R}^{base}$ | 基础剩余流量预测，也就是不做校准时的预测。 |
| $c$ | 校准系数。大于 1 表示最近流量偏高，要上调预测；小于 1 表示最近流量偏低，要下调预测。 |
| $\hat{R}$ | 最终输出的剩余流量预测。 |

直观理解：基础预测先回答“按历史规律今天还剩多少”，时间片校准再回答“最近流量状态是不是已经偏离历史规律”。

---

## 2. 时间片校准

时间片校准只看当前时间片之前最近 `{L}` 个连续时间片。这个窗口可以跨天，因此即使今天刚开始，也可以利用当前时刻之前最近的一段流量。

定义：

| 名词 | 含义 |
| --- | --- |
| $O$ | 最近 `{L}` 个时间片里的真实流量。 |
| $B$ | 最近 `{L}` 个时间片里的基础预测流量。 |
| $O / B$ | 真实流量相对基础预测的偏高或偏低程度。 |

如果 $O > B$，说明最近流量高于基础预测，剩余预测应该上调；如果 $O < B$，说明最近流量低于基础预测，剩余预测应该下调。

直接使用 $O / B$ 会比较抖，因此实际使用平滑后的校准系数：

$$
c
=
\left(
\frac{O+1}{B+1}
\right)^{
\frac{B}{B+\alpha}
}
$$

其中 $\alpha$ 是平滑强度参数。窗口里的基础预测流量 $B$ 越大，说明这个窗口的信息越充分，越相信最近流量；$B$ 越小，越保守，校准系数越接近 1。

校准后的预测为：

$$
\hat{R}
=
\hat{R}^{base}
\cdot
c
$$

---

## 3. 基础预测

基础预测的作用是：在不看当前在线观测的情况下，先给出每个任务、每个时间片的预估流量。

当前离线 SQL 中，基础时间片预测来自历史同任务、同时间片的平均流量：

$$
\hat{x}^{base}_{i,d,t}
=
\operatorname{Avg}
\left(
x_{i,d',t}
\right)
$$

其中 $i$ 表示任务，$d$ 表示日期，$t$ 表示时间片，$d'$ 表示历史日期。

基础剩余流量就是从当前时间片加到当天结束：

$$
\hat{R}^{base}_{i,d,k}
=
\sum_{t=k}^{47}
\hat{x}^{base}_{i,d,t}
$$

线上实现时，基础预测不一定要用历史均值。它也可以来自“模型预测全天流量 × 日内形状”，或直接来自时间片级流量模型。后续校准逻辑只要求能拿到两个值：

* 当前时间片到当天结束的基础剩余预测 $\hat{R}^{base}$。
* 最近 `{L}` 个时间片里的基础预测流量 $B$。

---

## 4. 平滑公式

平滑的目的是避免低流量窗口带来的比例抖动。

如果直接令：

$$
c
=
\frac{O}{B}
$$

会有几个问题：

* 低流量窗口里，少量请求变化就会让比例剧烈变化。
* 当 $B$ 很小时，比例容易被放大。
* 最近窗口的偏差不一定会完整延续到后续剩余时间。

因此使用：

$$
c
=
\left(
\frac{O+1}{B+1}
\right)^{
\frac{B}{B+\alpha}
}
$$

这个公式可以拆成两层：

1. $(O+1)/(B+1)$ 表示最近真实流量和基础预测的比例，加 1 是为了减少低流量时的极端值。
2. $B/(B+\alpha)$ 表示相信最近窗口的程度。$B$ 越大，权重越接近 1；$B$ 越小，权重越接近 0。

当窗口流量足够多时，校准会更接近最近真实比例；当窗口流量很少时，校准会更接近 1，也就是更相信基础预测。
