# 流量预测与校准算法设计文档

## 0. 先看结论

这个方案要解决的问题很简单：

> 在每个时间片开始前，预测某个任务今天从当前时间片到当天结束还会有多少流量。

当前离线 SQL 已验证：在基础剩余流量预测之上，利用“已经发生的流量”做在线校准，可以明显降低误差。最近 `{L}` 个时间片校准比当天累计校准略优，说明短期流量状态对剩余流量预测有帮助。

| 方案 | 离线评估值 |
| --- | ---: |
| 老方法 / 未校准基础预测 | `0.90` |
| 当天已有流量累计校准 | `0.75` |
| 最近 `{L}` 个时间片校准 | `0.70` |

推荐把“最近 `{L}` 个时间片校准”作为当前主方案继续验证。这里的 `{L}` 是占位符，后续按实验窗口长度替换。

需要特别说明：SQL 字段名里沿用了 `*_wape`，但当前结果不是严格经典 WMAPE / WAPE。当前评估值可以理解为“用当前时间片流量加权的相对误差”，后文统一称为 `Score`。

---

## 1. 方案在做什么

当前数据把一天切成 `0-47` 共 48 个时间片。本文中的“流量”就是 SQL 里的 `req_cnt`。

核心思路是：

1. 先用历史数据给每个任务、每个时间片做一条基础预测曲线。
2. 到当前时间片时，看最近已经发生的流量，比基础预测偏高还是偏低。
3. 把这个偏差变成一个平滑后的校准系数，用它放大或缩小基础剩余流量预测。

最核心的公式只有一个：

$$
\hat{R}
=
\hat{R}^{base}
\cdot
c
$$

其中：

| 名词 | 含义 |
| --- | --- |
| $\hat{R}^{base}$ | 基础剩余流量预测，也就是不做在线校准时的预测。 |
| $c$ | 校准系数。大于 1 表示最近流量偏高，要上调预测；小于 1 表示最近流量偏低，要下调预测。 |
| $\hat{R}$ | 最终输出给下游使用的剩余流量预测。 |

校准系数来自一个已观测窗口：

$$
c
=
\left(
\frac{O+1}{B+1}
\right)^{
\frac{B}{B+\alpha}
}
$$

其中：

| 名词 | 含义 |
| --- | --- |
| $O$ | 已观测窗口里的真实流量。 |
| $B$ | 同一个窗口里，基础预测认为应该发生的流量。 |
| $(O+1)/(B+1)$ | 真实流量相对基础预测的比例。加 1 是为了避免低流量时比例爆炸。 |
| $B/(B+\alpha)$ | 平滑权重。窗口里基础预测流量越大，越相信这次观测；越小，越保守。 |
| $\alpha$ | 平滑强度参数。越大越保守，越小越敏感。 |

读到这里就能理解整体方案：基础预测先给一个“今天剩下多少”的答案，再用最近已经发生的流量把它校准一下。

---

## 2. 两种校准窗口

当前文档讨论两种窗口。两者公式完全一样，区别只是 $O$ 和 $B$ 取哪段已发生流量。

| 方案 | 已观测窗口 | 直觉 | 当前结果 |
| --- | --- | --- | ---: |
| 当天累计校准 | 今天从 `0` 到当前时间片之前 | 看今天整体比基础预测高还是低 | `0.75` |
| 最近 `{L}` 个时间片校准 | 当前时间片之前最近 `{L}` 个连续时间片，可跨天 | 看最近一段时间的短期状态 | `0.70` |

当天累计校准更稳定，但早期可用信息少，而且今天早些时候的偏差不一定代表当前状态。

最近 `{L}` 个时间片校准更关注短期状态。它可以跨天取窗口，因此即使当天刚开始，也能用到当前时刻之前最近的一段流量。当前实验里它优于当天累计校准，说明最近窗口对剩余流量有额外解释力。

## 3. 数据样本与评估口径

离线 SQL 以 `task_id + pt_d + time_slice` 作为一个评估样本。

预测目标是当前时间片到当天结束的真实剩余流量：

$$
R_{i,d,k}
=
\sum_{t=k}^{47} x_{i,d,t}
$$

其中：

| 字段 / 符号 | 含义 |
| --- | --- |
| `task_id` / $i$ | 任务。 |
| `pt_d` / $d$ | 日期。 |
| `time_slice` / $k$ | 当前时间片。 |
| `req_cnt` / $x_{i,d,k}$ | 当前时间片真实流量。 |
| `actual_remain` / $R_{i,d,k}$ | 从当前时间片到当天结束的真实剩余流量。 |
| `base_remain` / $\hat{R}^{base}_{i,d,k}$ | 基础剩余流量预测。 |

当前 SQL 主要过滤条件是：

* `actual_remain > 0`
* `base_remain > 0`
* `req_cnt > 0`
* `base_cum_today > 0`
* `min_hist_day_cnt >= 3`

当前评估口径是：

$$
\text{Score}
=
\frac{
\sum_s q_s \cdot \frac{|\hat{R}_s - R_s|}{R_s}
}{
\sum_s q_s
}
$$

其中 $s=(i,d,k)$ 表示一个样本，$q_s$ 是该样本当前时间片的 `req_cnt`。也就是说，这个指标会让当前时间片流量更大的样本权重更高，同时看的是相对误差。

它和经典 WAPE 的区别是：

* 经典 WAPE 通常是 `sum(abs_error) / sum(actual)`。
* 当前 SQL 是 `sum(req_cnt * abs_error / actual_remain) / sum(req_cnt)`。

因此对外描述时，不建议直接说“WMAPE 从 0.90 降到 0.70”，更准确的说法是：

> 当前 SQL 评估口径下，`req_cnt` 加权相对误差从 `0.90` 降到 `0.70`。

从优化目标看，当前指标更接近以 `req_cnt / actual_remain` 加权的绝对误差。因此它不能简单解释为普通 P50 最优；更合适的理解是：在当前加权相对误差口径下，在线校准显著改善了基础预测。

## 4. 基础预测

基础预测的作用是：在不看当前在线观测的情况下，先给出每个任务、每个时间片应该有多少流量。

当前离线 SQL 中，基础时间片预测来自历史同任务、同时间片的平均流量：

$$
\hat{x}^{base}_{i,d,t}
=
\operatorname{Avg}
\left(
x_{i,d',t}
\right)
$$

其中 $d'$ 是评估日前一段历史窗口内的历史日期。

基础剩余流量就是从当前时间片加到当天结束：

$$
\hat{R}^{base}_{i,d,k}
=
\sum_{t=k}^{47}
\hat{x}^{base}_{i,d,t}
$$

在线工程化时，基础预测不一定要用历史均值。它也可以来自“模型预测全天流量 × 日内形状”，或直接来自时间片级流量模型。后续校准逻辑只要求能拿到两个东西：

* 当前时间片到当天结束的基础剩余预测 $\hat{R}^{base}$。
* 已观测窗口内的基础预测流量 $B$。

## 5. 两种校准公式

### 5.1 当天累计校准

当天累计校准使用今天已经发生的时间片：

$$
O^{today}_{i,d,k}
=
\sum_{t=0}^{k-1}
x_{i,d,t}
$$

对应的基础预测累计流量为：

$$
B^{today}_{i,d,k}
=
\sum_{t=0}^{k-1}
\hat{x}^{base}_{i,d,t}
$$

校准系数为：

$$
c^{today}_{i,d,k}
=
\left(
\frac{
O^{today}_{i,d,k}+1
}{
B^{today}_{i,d,k}+1
}
\right)^{
\frac{
B^{today}_{i,d,k}
}{
B^{today}_{i,d,k}+\alpha
}
}
$$

校准后的预测为：

$$
\hat{R}^{today}_{i,d,k}
=
\hat{R}^{base}_{i,d,k}
\cdot
c^{today}_{i,d,k}
$$

### 5.2 最近 `{L}` 个时间片校准

最近 `{L}` 个时间片校准使用当前时间片之前最近 `{L}` 个连续时间片。窗口可以跨天。

记 $g(d,k)$ 为日期和时间片展开后的全局时间片序号。窗口真实流量为：

$$
O^{win}_{i,d,k,\{L\}}
=
\sum_{g(d',t) \in [g(d,k)-L,\,g(d,k)-1]}
x_{i,d',t}
$$

对应基础预测窗口流量为：

$$
B^{win}_{i,d,k,\{L\}}
=
\sum_{g(d',t) \in [g(d,k)-L,\,g(d,k)-1]}
\hat{x}^{base}_{i,d',t}
$$

校准系数为：

$$
c^{win}_{i,d,k,\{L\}}
=
\left(
\frac{
O^{win}_{i,d,k,\{L\}}+1
}{
B^{win}_{i,d,k,\{L\}}+1
}
\right)^{
\frac{
B^{win}_{i,d,k,\{L\}}
}{
B^{win}_{i,d,k,\{L\}}+\alpha
}
}
$$

校准后的剩余流量预测为：

$$
\hat{R}^{win}_{i,d,k,\{L\}}
=
\hat{R}^{base}_{i,d,k}
\cdot
c^{win}_{i,d,k,\{L\}}
$$

## 6. 为什么要做平滑

最直接的校准方式是用真实流量除以基础预测：

$$
r
=
\frac{O}{B}
$$

然后令 $c=r$。但这样会很不稳定：

* 早期或低流量窗口里，$O$ 和 $B$ 都很小，比例容易剧烈波动。
* 少量请求变化就可能让比例从很小跳到很大。
* 当前窗口的偏差不一定会完整延续到后续剩余时间。

所以当前方案使用：

$$
c
=
\left(
\frac{O+1}{B+1}
\right)^{
\frac{B}{B+\alpha}
}
$$

这个公式可以拆成两层理解：

1. $(O+1)/(B+1)$ 是“观测流量 / 基础预测流量”的比例，加 1 用来避免低流量爆炸。
2. $B/(B+\alpha)$ 是“相信观测多少”的权重。窗口基础预测流量越大，观测越可靠，权重越接近 1；窗口基础预测流量越小，权重越接近 0，校准系数也越接近 1。

如果从概率角度解释，可以把真实校准因子记为 $Z$：

$$
R
=
\hat{R}^{base}
\cdot
Z
$$

在 log 空间里，令 $u=\log Z$，把窗口比例 $\log((O+1)/(B+1))$ 看成对 $u$ 的带噪声观测。窗口里的基础预测流量 $B$ 越大，观测噪声越小，就越应该相信这个比例。这样可以得到同样的工程化权重：

$$
w
=
\frac{B}{B+\alpha}
$$

最终校准系数就是：

$$
c
=
\left(
\frac{O+1}{B+1}
\right)^w
$$

这就是当天累计校准和最近 `{L}` 个时间片校准共同使用的形式。
