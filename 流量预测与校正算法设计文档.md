# 流量预测与校正算法设计文档

本文档基于当前项目中的《流量预测与校正方案.txt》和离线验证 SQL，整理广告 MPC 出价场景下的当天剩余流量预测与在线校正方案。文档目标是说明该模块为什么需要、优化目标是什么、理论最优形式是什么、当前 MVP 为什么采用“历史基础预测 + 当天 shrinkage 校正”，以及离线 SQL 如何验证该方案。

本文只讨论算法设计和离线验证口径，不包含工程实现代码。

---

# 1. 背景与问题定义

当前广告系统使用 MPC 出价。MPC 在每个时间片需要根据未来一段时间的预估状态做决策，包括未来转化、未来消耗、预算约束、成本约束和 ROI 约束。因此，在当天时间点 `k`，系统需要预估任务 `i` 从当前时间点之后到当天结束的剩余流量。

如果剩余流量被高估，MPC 可能认为后续仍有充足机会，从而当前出价过于保守，导致后续预算花不出去或转化机会损失。如果剩余流量被低估，MPC 可能认为后续机会不足，从而当前出价过激，造成预算提前消耗、成本或 ROI 约束变差。因此，剩余流量预测是 MPC 控制链路中的关键输入。

当前任务级流量预估存在以下难点：

* 任务级流量日间波动大，同一 `task_id` 在不同日期的流量规模可能差异明显。
* 单纯依赖历史数据预测未来流量不稳定，尤其在任务生命周期、预算、出价或投放状态变化时。
* 当天已观测流量可能被预算耗尽、停投、pacer 压制、状态异常、数据延迟等因素污染，不能直接等同于自然流量变化。
* 直接预测所有未来时间片的流量曲线较难训练，目标维度高，且不同任务尺度差异大。
* 校正系数如果依赖人工设置，很难兼顾不同任务、不同时间片和不同流量规模。
* 预测误差会传导到 MPC 的预算控制、成本控制和 ROI 控制，影响最终投放效果。

当前模块的预测目标是：

> 在当天日期 `d`、当前时间片 `k`，预测任务 `i` 从 `k` 之后到当天结束的剩余流量。

在当前离线 SQL 中，`time_slice <= k` 被视为已观测部分，`time_slice > k` 被视为剩余部分。因此本文默认：

$$
R_{i,d,k} = \sum_{t=k+1}^{47} x_{i,d,t}
$$

其中 `time_slice` 范围为 `0-47`。如果线上调用发生在时间片尚未完全结束时，可以根据实时可用数据重新定义边界，但需要保持训练、验证和线上使用口径一致。

# 2. 变量定义

| 符号 / 字段 | 含义 |
| --- | --- |
| `task_id` / `i` | 广告任务或投放任务。 |
| `d` | 日期。 |
| `t` | 时间片，当前离线口径为 `0-47`。 |
| `k` | 当前预测时间片。 |
| $x_{i,d,t}$ | 任务 `i` 在日期 `d`、时间片 `t` 的真实流量。当前 SQL 中对应聚合后的 `req_cnt`。 |
| $C_{i,d,k}$ | 当天截至当前时间片 `k` 的累计真实流量。 |
| $R_{i,d,k}$ | 当前时间片 `k` 之后的真实剩余流量。 |
| $\hat{R}_{i,d,k}$ | 模型预测的剩余流量。 |
| $x^{base}_{i,d,t}$ | 基于历史数据得到的时间片级基础预测流量。当前 MVP 中对应 `base_req_cnt`。 |
| $C^{base}_{i,d,k}$ | 基础预测截至当前时间片 `k` 的累计预测流量。当前 SQL 中对应 `base_cum`。 |
| $R^{base}_{i,d,k}$ | 基础预测在当前时间片 `k` 之后的剩余预测流量。当前 SQL 中对应 `base_remain`。 |
| $r_{i,d,k}$ | 当天累计实际 / 累计预测比例，用于刻画当天已观测偏差。 |
| $c_{i,d,k}$ | 校正系数，用于把基础剩余预测校正为最终剩余预测。 |
| $Z_{i,d,k}$ | 真实校正因子，满足 $R_{i,d,k}=R^{base}_{i,d,k}Z_{i,d,k}$。 |
| $n_{i,d,k}$ | 当前已经观测到的有效样本量或有效累计流量。当前 MVP 近似使用 `base_cum`。 |
| $\alpha$ | shrinkage 平滑参数，表示历史先验强度。当前离线测试先固定为 `1000.0`。 |
| $\mathcal{F}_{i,d,k}$ | 当前预测时可用的信息集合，包括历史流量、当天已观测流量、任务状态、预算、pacer 状态等。当前 MVP 只使用其中的一部分。 |

核心累计量定义如下：

$$
C_{i,d,k} = \sum_{t=0}^{k} x_{i,d,t}
$$

$$
C^{base}_{i,d,k} = \sum_{t=0}^{k} x^{base}_{i,d,t}
$$

$$
R^{base}_{i,d,k} = \sum_{t=k+1}^{47} x^{base}_{i,d,t}
$$

当前 MVP 中，当天累计比例使用加 1 平滑：

$$
r_{i,d,k} =
\frac{C_{i,d,k}+1}{C^{base}_{i,d,k}+1}
$$

加 1 平滑的作用是降低早期小流量和零流量情况下的比例爆炸风险。

# 3. 优化目标与指标

当前定义的评估指标是“按真实剩余流量大小加权的 MAPE”。对某个当前时间片 `k`，若省略日期和时间片下标，任务 `i` 的真实剩余流量为 $R_i$，预测剩余流量为 $\hat{R}_i$，则普通 MAPE 的单任务误差为：

$$
\frac{|\hat{R}_i - R_i|}{R_i}
$$

如果按真实剩余流量 $R_i$ 加权，整体指标为：

$$
\text{Weighted MAPE}
=
\frac{
\sum_i R_i \cdot \frac{|\hat{R}_i - R_i|}{R_i}
}{
\sum_i R_i
}
$$

化简可得：

$$
\text{Weighted MAPE}
=
\frac{
\sum_i |\hat{R}_i - R_i|
}{
\sum_i R_i
}
$$

因此，该指标本质上接近 WAPE，也就是用真实流量总和归一化的 MAE：

$$
\text{WAPE}
=
\frac{\text{绝对误差总和}}{\text{真实流量总和}}
$$

该指标的优点包括：

* 大流量任务对整体误差的影响更合理，符合 MPC 对整体剩余流量规模的关注。
* 避免普通 MAPE 在小流量任务或接近零流量任务上爆炸。
* 与预算、消耗、转化预测等总量型控制目标更一致。
* 在离线验证中易于聚合，可以直接比较基础预测和校正后预测的总体误差。

需要注意的是，在 $R_i=0$ 或非常小时，普通 MAPE 不稳定。因此当前 SQL 过滤了 `actual_remain > 0`，并使用 WAPE 聚合口径评估整体误差。

在 MAE / WAPE 型目标下，如果将分母视为给定或相对稳定，则优化目标等价于最小化绝对误差期望：

$$
\min_{\hat{R}_{i,d,k}}
\mathbb{E}
\left[
|\hat{R}_{i,d,k} - R_{i,d,k}|
\mid
\mathcal{F}_{i,d,k}
\right]
$$

该目标下，理论最优点预测不是条件均值，而是条件中位数：

$$
\hat{R}^{*}_{i,d,k}
=
\operatorname{Median}
\left(
R_{i,d,k}
\mid
\mathcal{F}_{i,d,k}
\right)
$$

也就是说，在当前指标口径下，最优预测应理解为：

> 给定历史信息、当天观测信息和当前状态后的后验 P50。

---

# 4. 理论推导：为什么是后验 P50

令 $Y=R_{i,d,k}$ 表示某个任务在当前时间片后的真实剩余流量，令 $a=\hat{R}_{i,d,k}$ 表示点预测。在给定信息集合 $\mathcal{F}_{i,d,k}$ 下，需要最小化：

$$
g(a)
=
\mathbb{E}
\left[
|a-Y|
\mid
\mathcal{F}_{i,d,k}
\right]
$$

将条件分布记为 $F(y)=P(Y \le y \mid \mathcal{F}_{i,d,k})$。对绝对损失而言，$g(a)$ 的次梯度为：

$$
\partial g(a)
=
P(Y \le a \mid \mathcal{F}_{i,d,k})
-
P(Y \ge a \mid \mathcal{F}_{i,d,k})
$$

当 $0$ 落在次梯度集合中时，$a$ 为最优解。该条件等价于：

$$
P(Y \le a \mid \mathcal{F}_{i,d,k}) \ge \frac{1}{2}
$$

且：

$$
P(Y \ge a \mid \mathcal{F}_{i,d,k}) \ge \frac{1}{2}
$$

满足该条件的点就是条件中位数。因此：

$$
\hat{R}^{*}_{i,d,k}
=
\operatorname{Median}
\left(
R_{i,d,k}
\mid
\mathcal{F}_{i,d,k}
\right)
$$

这就是后验 P50 的来源。

如果严格考虑全局 WAPE：

$$
\frac{\sum_i |\hat{R}_i - R_i|}{\sum_i R_i}
$$

分母 $\sum_i R_i$ 本身也是随机变量。此时理论上并不是简单地对每个任务独立求普通条件中位数，而是会引入与全局真实剩余流量总和相关的权重，最优解可以理解为带权条件中位数。

工程上，如果评估集合足够大，且全局剩余流量总和相对稳定，分母波动对单任务最优点的影响通常可以近似忽略。此时使用普通条件中位数作为目标，是一个合理且易落地的近似。

# 5. “历史预测 + 当天校正”的理论解释

当前方案采用：

$$
\hat{R}_{i,d,k}
=
R^{base}_{i,d,k}
\cdot
c_{i,d,k}
$$

其中 $R^{base}_{i,d,k}$ 是历史基础预测得到的剩余流量，$c_{i,d,k}$ 是根据当天已观测流量得到的校正系数。

该结构不是简单的经验规则，可以理解为一种后验估计近似：

* 历史基础预测提供先验，描述在没有当天观测时，任务在未来时间片的典型流量形态。
* 当天已发生流量提供观测，描述当天相对于历史先验的偏高或偏低。
* 校正系数是后验校正因子的估计，用于把历史先验更新为当天条件下的预测。

定义真实校正因子 $Z_{i,d,k}$：

$$
R_{i,d,k}
=
R^{base}_{i,d,k}
\cdot
Z_{i,d,k}
$$

如果 $R^{base}_{i,d,k}>0$，则预测误差可以写成：

$$
|\hat{R}_{i,d,k} - R_{i,d,k}|
=
R^{base}_{i,d,k}
\cdot
|c_{i,d,k} - Z_{i,d,k}|
$$

在 MAE / WAPE 型目标下，给定信息集合 $\mathcal{F}_{i,d,k}$，最优校正系数应为真实校正因子的条件中位数：

$$
\hat{Z}^{*}_{i,d,k}
=
\operatorname{Median}
\left(
Z_{i,d,k}
\mid
\mathcal{F}_{i,d,k}
\right)
$$

因此：

$$
\hat{R}^{*}_{i,d,k}
=
R^{base}_{i,d,k}
\cdot
\operatorname{Median}
\left(
Z_{i,d,k}
\mid
\mathcal{F}_{i,d,k}
\right)
$$

这说明“历史预测 + 当天校正”可以是合理结构，但前提是校正项估计的是后验校正因子的 P50，而不是手动固定比例或无约束地直接使用当天累计比例。

从全局最优角度看，真正的目标仍然是：

$$
\operatorname{Median}
\left(
R_{i,d,k}
\mid
\mathcal{F}_{i,d,k}
\right)
$$

“历史预测 + 当天校正”只是使用 $R^{base}_{i,d,k}$ 和当天累计比例等有限信息来近似这个后验中位数。当这些信息足够表达剩余流量的主要变化时，该结构效果会较好；当存在预算耗尽、pacer 压制、停投、状态异常等未建模因素时，该结构会存在偏差。

# 6. 校正系数的 shrinkage 推导

一个直观做法是直接使用当天累计实际 / 预测比例作为校正系数：

$$
r_{i,d,k}
=
\frac{C^{obs}_{i,d,k}}{C^{base}_{i,d,k}}
$$

然后令：

$$
\hat{R}_{i,d,k}
=
R^{base}_{i,d,k}
\cdot
r_{i,d,k}
$$

但直接使用该比例通常不稳，原因包括：

* 当前时间片较早时，已观测流量少，比例方差很大。
* 低流量任务的累计比例容易因少量请求产生极端波动。
* 当天观测可能被预算、停投、pacer 压制、状态异常或数据延迟污染。
* 当前累计偏差不一定会在未来剩余时间段保持同样比例。
* 基础累计预测很小时，比例可能爆炸。

因此，校正系数需要 shrinkage：当当天证据不足时，更相信历史基础预测；当当天证据充分时，再逐步增强校正。

在 log 空间中定义真实校正因子：

$$
u_{i,d,k}
=
\log Z_{i,d,k}
$$

设历史先验为：

$$
u_{i,d,k}
\sim
\mathcal{N}(m_0, \sigma_0^2)
$$

当天累计比例的 log 观测为：

$$
\ell_{i,d,k}
=
\log r_{i,d,k}
$$

并假设：

$$
\ell_{i,d,k}
\mid
u_{i,d,k}
\sim
\mathcal{N}(u_{i,d,k}, \sigma_{obs}^2)
$$

根据正态-正态共轭，后验仍为正态分布，其均值为：

$$
m_{post}
=
\frac{
\sigma_{obs}^2 m_0 + \sigma_0^2 \ell_{i,d,k}
}{
\sigma_0^2 + \sigma_{obs}^2
}
$$

写成加权形式：

$$
m_{post}
=
(1-w)m_0 + w \log r_{i,d,k}
$$

其中：

$$
w
=
\frac{\sigma_0^2}{\sigma_0^2 + \sigma_{obs}^2}
$$

由于后验分布为正态分布，后验均值也等于后验中位数。若历史先验中心为 1，即 $m_0=0$，则后验校正因子为：

$$
\hat{Z}_{i,d,k}
=
\exp(m_{post})
=
r_{i,d,k}^{w}
$$

最终预测为：

$$
\hat{R}_{i,d,k}
=
R^{base}_{i,d,k}
\cdot
r_{i,d,k}^{w}
$$

进一步，如果当天观测噪声方差与当前有效观测量成反比：

$$
\sigma_{obs}^2
\propto
\frac{1}{n_{i,d,k}}
$$

则可以得到工程形式：

$$
w_{i,d,k}
=
\frac{n_{i,d,k}}{n_{i,d,k}+\alpha}
$$

其中 $\alpha$ 可以理解为历史先验强度。于是当前 MVP 公式为：

$$
\hat{R}_{i,d,k}
=
R^{base}_{i,d,k}
\cdot
r_{i,d,k}^{\frac{n_{i,d,k}}{n_{i,d,k}+\alpha}}
$$

在当前离线 SQL 中，使用：

$$
n_{i,d,k}
\approx
C^{base}_{i,d,k}
$$

并且：

$$
r_{i,d,k}
=
\frac{C_{i,d,k}+1}{C^{base}_{i,d,k}+1}
$$

因此 SQL 中的 MVP 预测为：

$$
\hat{R}_{i,d,k}
=
R^{base}_{i,d,k}
\cdot
\left(
\frac{C_{i,d,k}+1}{C^{base}_{i,d,k}+1}
\right)^{
\frac{C^{base}_{i,d,k}}{C^{base}_{i,d,k}+\alpha}
}
$$

$\alpha$ 的含义如下：

* $\alpha$ 可以理解为历史先验强度。
* $\alpha$ 越大，当天观测需要更多样本才会形成强校正。
* $\alpha$ 越小，模型越快相信当天观测。
* 当前离线测试中 $\alpha$ 先固定为 `1000.0`。
* 后续可以通过历史回放，在不同任务规模、不同时间片、不同业务分层上调参。

# 7. 当前离线验证方案

当前项目已有一个可运行的 Hive SQL 离线验证方案，文件为 `offline_mvp.sql`。该 SQL 使用已有表字段：

* `task_id`
* `pt_d`
* `req_cnt`
* `time_slice`，范围 `0-47`

验证思路是：对于评估日期 `d`，把它当成线上“当天”；对于每个当前时间片 `k`，只使用 `d` 之前的数据构造历史基础预测，再使用 `d` 当天 `time_slice <= k` 的真实流量模拟线上已观测流量，预测 `time_slice > k` 的剩余流量，并与真实剩余流量对比。

具体计算流程如下。

第一步，构造评估日数据 `a`。SQL 从 `your_table` 中读取评估日期范围内的数据，按 `task_id + pt_d + time_slice` 聚合 `req_cnt`。这部分代表被模拟成“当天”的真实流量曲线。

第二步，构造历史数据 `h`。SQL 读取评估开始日期往前 7 天到评估结束日期之间的数据，同样按 `task_id + pt_d + time_slice` 聚合 `req_cnt`。在 join 时要求：

* `a.task_id = h.task_id`
* `a.time_slice = h.time_slice`
* `h.pt_d < a.pt_d`
* `h.pt_d >= a.pt_d - 7 天`

因此，对于评估日 `d` 的某个任务和时间片，基础预测只使用该任务过去 7 天同一时间片的历史流量。

第三步，计算时间片级基础预测：

$$
x^{base}_{i,d,t}
=
\operatorname{AVG}
\left(
x_{i,d',t}
\right),
\quad
d-7 \le d' < d
$$

SQL 中对应字段为 `base_req_cnt`。同时，`hist_day_cnt` 记录该时间片实际匹配到的历史天数。

第四步，计算 `min_hist_day_cnt`。SQL 对每个 `task_id + pt_d` 取所有时间片 `hist_day_cnt` 的最小值，用于过滤历史数据不足的样本。当前过滤条件为：

```sql
min_hist_day_cnt >= 3
```

也就是说，进入评估的任务日期至少需要具备相对完整的历史基础预测。

第五步，使用窗口函数计算当前时间片 `k` 下的累计和剩余量：

* `obs_cum`：当天真实流量从 `0` 到 `k` 的累计和。
* `base_cum`：基础预测从 `0` 到 `k` 的累计和。
* `actual_remain`：当天真实流量从 `k+1` 到当天结束的累计和。
* `base_remain`：基础预测从 `k+1` 到当天结束的累计和。

第六步，生成两种预测：

基础预测不做校正：

$$
\hat{R}^{base}_{i,d,k}
=
R^{base}_{i,d,k}
$$

SQL 中对应：

```sql
base_remain AS base_pred_remain
```

MVP shrinkage 校正预测：

```sql
base_remain
*
POW(
    (obs_cum + 1.0) / (base_cum + 1.0),
    base_cum / (base_cum + ${hivevar:alpha})
) AS mvp_pred_remain
```

该公式对应理论形式：

$$
\hat{R}
=
R_{base}
\cdot
r^w
$$

其中：

$$
r
=
\frac{obs\_cum + 1}{base\_cum + 1}
$$

$$
w
=
\frac{base\_cum}{base\_cum + \alpha}
$$

第七步，过滤无效样本。当前 SQL 使用以下条件：

* `time_slice BETWEEN 0 AND 46`：最后一个时间片没有未来剩余流量，因此不评估。
* `min_hist_day_cnt >= 3`：保证历史基础预测有至少 3 天支撑。
* `actual_remain > 0`：避免真实剩余流量为 0 的样本影响 WAPE 口径。
* `base_remain > 0`：保证基础剩余预测为正。
* `base_cum > 0`：保证累计基础预测为正，使校正比例有意义。

第八步，聚合评估指标。SQL 输出：

* `sample_cnt`：参与评估的任务-日期-当前时间片样本数。
* `actual_remain_sum`：真实剩余流量总和。
* `base_pred_remain_sum`：基础预测剩余流量总和。
* `mvp_pred_remain_sum`：MVP 校正后预测剩余流量总和。
* `base_abs_error`：基础预测绝对误差总和。
* `mvp_abs_error`：MVP 校正预测绝对误差总和。
* `base_wape`：基础预测 WAPE。
* `mvp_wape`：MVP 校正预测 WAPE。
* `relative_error_reduction`：MVP 相对基础预测的绝对误差下降比例。

关键字段含义如下：

| 字段 | 含义 |
| --- | --- |
| `base_req_cnt` | 历史 7 天同 `task_id`、同 `time_slice` 的平均流量，作为时间片级基础预测。 |
| `obs_cum` | 当天截至当前时间片的累计真实流量。 |
| `base_cum` | 基础预测截至当前时间片的累计预测流量。 |
| `actual_remain` | 当前时间片之后的真实剩余流量。 |
| `base_remain` | 当前时间片之后的基础预测剩余流量。 |
| `mvp_pred_remain` | 经过 shrinkage 校正后的预测剩余流量。 |
| `alpha` | 校正平滑参数，当前固定为 `1000.0`。 |
| `base_wape` | 不校正的历史基础预测误差。 |
| `mvp_wape` | 校正后的预测误差。 |
| `relative_error_reduction` | 相对误差下降比例，值越大表示校正带来的收益越明显。 |

# 8. 当前实验的意义与局限

当前 SQL 主要验证以下问题：

* 最近 7 天同时间片均值作为历史基础预测是否具备有效性。
* 当天累计实际 / 预测偏差是否能改善剩余流量预测。
* shrinkage 校正是否优于完全不校正的基础预测。
* 在 $\alpha=1000$ 时，MVP 校正是否能降低全局 WAPE。
* 当前公式是否能在任务级、时间片级回放中稳定地产生收益。

当前实验的意义在于，它把线上预测问题转化成离线可回放问题：把历史某天当成“当天”，只使用该天之前的历史数据和该天截至当前时间片的已发生数据，模拟线上实时预测过程。这种验证方式可以较低成本地判断“历史基础预测 + 当天 shrinkage 校正”是否值得进入后续工程化。

同时，当前实验也有明显局限：

* 基础预测只使用最近 7 天同时间片平均，基础模型较简单，无法显式建模星期、节假日、趋势、行业、大盘或任务状态变化。
* 当前没有过滤停投、预算耗尽、pacer 压制、审核异常、出价策略变化、数据延迟等脏数据。
* 当前没有区分大盘整体波动和任务自身波动。如果所有任务当天整体流量偏高，任务级比例和大盘比例应被区别使用。
* 当前 $\alpha$ 是固定值，不同任务规模、不同时间片、不同历史稳定性没有分层。
* 当前只输出点预测，没有输出不确定性区间或分位数。
* 当前离线数据只包含 `task_id`、`pt_d`、`req_cnt`、`time_slice`，缺少任务状态、预算状态、pacer 状态、出价、定向、媒体、行业等控制变量。
* 当前验证指标是流量误差，还没有直接验证对 MPC 预算、ROI、成本和消耗节奏的影响。
* 当前假设当天前半段累计偏差对后续剩余时间仍有参考价值，但真实场景中可能存在上午异常、下午恢复，或某些时段局部异常。

因此，当前 MVP 更适合作为第一阶段验证方案，而不是最终最优算法。它的价值在于用简单、可解释、可回放的形式验证当天在线校正是否存在显著收益。

# 9. 风险与注意事项

该方案落地时需要重点关注以下风险。

第一，不要把投放系统控制后的低流量误判为自然流量下降。如果任务因为预算接近耗尽、pacer 压制、停投、审核异常或出价降低而流量下降，直接用当天累计比例校正未来流量，会把控制动作造成的低流量错误外推到未来。

第二，不要对低流量任务做过强校正。低流量任务的累计比例方差很大，少量请求变化就可能导致比例大幅波动。对这类任务应依赖 shrinkage、最小样本量、上下界截断或分层参数。

第三，不要长期对所有任务使用同一个校正强度。不同任务的历史稳定性、流量规模、预算约束和投放状态差异很大，固定 $\alpha$ 只能作为 MVP 验证口径，后续应考虑按任务规模、历史波动、行业或时间片分层。

第四，不要只看点预测误差，也要看 MPC 控制结果。WAPE / MAE 最优对应 P50，但 MPC 的真实损失可能是不对称的。如果低估剩余流量和高估剩余流量对预算、ROI、成本的影响不同，MPC 可能需要 P60、P70 或 P80 等更保守或更激进的分位数输入。

第五，不要只使用普通 MAPE。普通 MAPE 对小流量任务极其敏感，容易被低流量样本主导，不能很好反映 MPC 对整体流量和消耗规模的关注。当前使用的加权 MAPE / WAPE 更适合作为第一阶段指标。

第六，要区分自然流量、可参与机会量、实际曝光量和实际消耗量。当前 `req_cnt` 代表的流量口径需要在业务上明确。如果它已经受投放控制影响，则它不完全等价于自然机会量；如果 MPC 需要的是可参与机会量或未来曝光量，则需要进一步校准数据口径。

第七，需要关注校正上下界。即使使用 shrinkage，当累计比例极端时，仍可能产生过大或过小的预测。工程落地时建议对 $r$ 或最终校正系数 $c$ 设置合理上下界，并按任务规模分层评估影响。

# 10. 总结与当前结论

当前方案可以归纳为以下结论：

* 当前指标“按真实剩余流量大小加权的 MAPE”可以化简为绝对误差总和除以真实流量总和，本质接近 WAPE / MAE。
* 在 MAE / WAPE 型目标下，如果分母相对稳定，理论最优点预测是后验 P50，也就是 $\operatorname{Median}(R_{i,d,k} \mid \mathcal{F}_{i,d,k})$。
* “历史预测 + 当天校正”不是天然最优，但可以被解释为后验估计的工程近似。
* 若定义 $R_{i,d,k}=R^{base}_{i,d,k}Z_{i,d,k}$，则最优校正系数应是真实校正因子 $Z_{i,d,k}$ 的后验中位数。
* 当前 MVP 使用 $R_{base} \cdot r^w$ 的 shrinkage 校正形式，避免直接使用当天累计比例带来的高方差问题。
* 当前权重 $w=base\_cum/(base\_cum+\alpha)$ 表示对当天观测的信任程度：当前累计基础流量越大，观测越充分，校正越强。
* 当前离线 SQL 可以验证 MVP 校正是否优于不校正的历史基础预测，核心比较指标是 `base_wape`、`mvp_wape` 和 `relative_error_reduction`。
* 当前实验仍受限于基础模型简单、脏数据未过滤、固定 $\alpha$、缺少状态变量和未验证 MPC 控制效果。

建议下一步按以下路径推进：

1. 先基于当前 SQL 固化多日期、多时间片、多任务规模分层的实验结果，确认 MVP 是否稳定降低 WAPE。
2. 对 $\alpha$ 做历史回放调参，至少按流量规模、时间片和历史波动程度分层评估。
3. 引入停投、预算耗尽、pacer 压制、状态异常、数据延迟等脏数据过滤或降权逻辑。
4. 增强基础预测模型，从简单 7 天均值升级为包含星期、节假日、趋势、大盘和任务状态的基础预测。
5. 在点预测稳定后，进一步评估分位数输出和 MPC 控制结果，确认预测误差下降是否能转化为预算、成本和 ROI 控制收益。

当前阶段的核心结论是：

> 当前 MVP 是一个合理的第一阶段方案。它用历史同时间片均值提供基础预测，用当天累计偏差提供在线观测，再通过 shrinkage 控制校正强度。该方案具备清晰的 MAE / WAPE 理论解释，也能通过现有 Hive SQL 做离线回放验证。后续重点应放在实验结果固化、参数分层、脏数据处理和更强基础预测上。
